Java高级 - 泛型

泛型列表

下边这段代码不会报错

List testList = new ArrayList();
for(int i = 0; i < 3; i++){
    testList.add("test");
}
testList.add(1);
System.out.println(testList);

因为没有给列表指定泛型类型

但是如果将其强制转换，也就是像下边这样

for(int i = 0; i < 4; i++){
    System.out.println((String)testList.get(i));
}

就会出现

java.lang.ClassCastException: java.lang.Integer cannot be cast to java.lang.String

的异常，因为Integer类型无法强制转换为String类型

但是下边这一段编译时就会报错

List<String> testList = new ArrayList<>();
for(int i = 0; i < 3; i++){
    testList.add("test");
}
testList.add(1);
System.out.println(testList);

因为已经指定了泛型类型，所输入的值就必须为String类型,就不会出现像上边参数类型不一致的问题。

泛型类

public class Generic <T>{
    private T test;

    public Generic(T test){
        this.test=test;
    }
}

这个类可以通过指定泛型参数T的类型指定传入参数的类型，从而提高适用性。

就像下边这样

Generic<String> testString = new Generic<>("test");
System.out.println(testString.getTest());
Generic<Integer> testInteger = new Generic<>(123);
System.out.println(testInteger.getTest());

泛型接口

public interface Generator <T>{
    T getInfo();
    void setInfo(T value);
}

如果它的实现类不指定泛型类型时，那么实现类中的方法就会自动转换位object如果指定类型的话，就会转换为自己指定的类型。

class testInterface1 implements Generator{
    @Override
    public Object getInfo() {
        return null;
    }
    @Override
    public void setInfo(Object value) {
    }
}

class testInterface3 <String> implements Generator<String>{
    @Override
    public String getInfo() {
        return null;
    }
    @Override
    public void setInfo(String value) {
    }
}

实现类后面的<String>可有可无

类型通配符

private static void show(Generic<Number> value){
    System.out.println(true);
}

如果使用下面这段代码时编译时就会报错

show(testInteger);

因为上边的testInteger是Generic<Integer>类型，但是show中的参数却是Generic<Number>类型，虽然Integer是Number的子类，但是仍然不能够将Generic<Integer>看做Generic<Number>的子类。

所以如果参数类型不确定时就可以使用下面通配符的方法：

private static void show1(Generic<?> value){
    System.out.println(true);
}

这样就可以使用上边的show(testInteger);代码

可以将?看做是所有类型的父类。

泛型方法

public static <T> void showArray(T[] inputArray){
    for(T input : inputArray){
        System.out.println(input);
    }
}

这个方法用于将传入的数组打印出来

<T>就是泛型方法的标志，当然也可以不只一个参数，例如<T,K>

这样就可以将多种类型的数组只使用一个方法打印出来：

String [] testMethodString = new String[]{"123", "456", "789"};
Integer [] testMethodInt = new Integer[]{1,2,3,4};
GenericTest.showArray(testMethodString);
GenericTest.showArray(testMethodInt);

下边这个方法可以指定参数的边界，也即传入的参数必须是指定类型的子类型或者父类型：

public static <T extends Comparable<T>> T max(T value1, T value2, T value3){
    T max = value1;
    if(value2.compareTo(max) > 0){
        max = value2;
    }
    if(value3.compareTo(max) > 0){
        max = value3;
    }
    return max;
}

使用时：

System.out.println("三个值中最大的是" + GenericTest.max(1,3,2));
System.out.println("三个值中最大的是" + GenericTest.max("test","a","z"));
