Java高级 - 多线程

概述

多线程是多个线程并发进行的技术，进而能够提高整体性能。而一个程序中，独立运行的程序片段叫做线程。

线程与进程的区别：

进程是操作系统资源调度的基本单位，而线程是任务的调度执行的基本单位

进程之间的切换开销较大

进程共享资源困难但是同步简单，而线程则相反。

线程分为四个状态：

  • 新建状态：使用new关键字创建出一个Thread对象后此线程就处于新建状态，直到调用start方法
  • 就绪状态：调用完start方法后，线程就处于就绪状态，在就绪队列中等待CPU的调度
  • 运行状态：如果就绪状态的线程获取到CPU资源，就可以运行run方法，此时Thread就处于运行状态，运行状态的线程可以转换到就绪状态，阻塞状态和死亡状态。
  • 死亡状态：一个线程完成任务或者遇到终止代码时就会进入到终止状态
  • 阻塞状态：线程中执行sleep，suspend等方法后线程就处于一种阻塞状态，在睡眠时间已到或者或者从新获取设备资源后就会到就绪状态。阻塞状态分为三种：
    • 等待阻塞：运行状态中的线程执行 wait() 方法，使线程进入到等待阻塞状态。
    • 同步阻塞：线程在获取 synchronized 同步锁失败(因为同步锁被其他线程占用)。
    • 其他阻塞：通过调用线程的 sleep() 或 join() 发出了 I/O 请求时，线程就会进入到阻塞状态。当sleep() 状态超时，join() 等待线程终止或超时，或者 I/O 处理完毕，线程重新转入就绪状态。

线程的优先级

优先级是一个1-10的整数，默认情况下线程会分配的优先级是5。

具有较高优先级的线程对程序更重要，并且应该在低优先级的线程之前分配处理器资源。但是，线程优先级不能保证线程执行的顺序，而且非常依赖于平台。

创建线程的三种方式

  1. 通过继承Thread类

通过重写run方法来实现这个线程的任务逻辑

public class ThreadTest extends Thread {
    private String threadName;
    private Thread thread;

    public ThreadTest(String threadName){
        super();
        this.threadName = threadName;
        System.out.println("create"+threadName);
    }

    @Override
    public void run() {
        System.out.println(threadName+ "running");
        try{
            for(int i = 0;i < 3; i++){
                System.out.println(threadName + " " + i);
                sleep(50);
            }
        }catch (InterruptedException e){
            e.printStackTrace();
        }
    }

    @Override
    public synchronized void start() {
        super.start();
        System.out.println("start"+threadName);
    }
}

然后在主方法中测试：

Thread thread1 = new ThreadTest("thread1");
thread1.start();
Thread thread2 = new ThreadTest("thread2");
thread2.start();
new ThreadTest("thread3").start();
  1. 通过实现Runnable接口，重写run方法

在run方法中实现线程的逻辑

public class RunnableTest implements Runnable{
    private String threadName;
    private Thread thread;

    public RunnableTest(String threadName){
        this.threadName = threadName;
        System.out.println("create "+ threadName);
    }

    @Override
    public void run() {
        System.out.println(threadName + " running");
        for(int i = 0; i < 3; i++){
            System.out.println(threadName + " " +i);
            try {
                Thread.sleep(50);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }

    public void start(){
        System.out.println(threadName + " start");
        if(thread == null){
            thread = new Thread(this, "thread");
            thread.start();
        }
    }
}

最后自己写了一个start方法用于给thread初始化并调用

而第一种方法中是重写的父类Thread中的方法

在main方法中测试：

RunnableTest runnable = new RunnableTest("Runnable thread1");
runnable.start();
RunnableTest runnableTest = new RunnableTest("Runnable thread2");
runnableTest.start();
  1. 通过Callable和Future来实现
public class CallableTest implements Callable {
    @Override
    public Object call() throws Exception {
        int i;
        for(i = 0; i < 10; i++){
            System.out.println(Thread.currentThread().getName()+ " " + i);
        }
        return i;
    }
}

注意：这个call方法是有返回值的

在主方法中调用时：

CallableTest test = new CallableTest();
FutureTask<Integer> task = new FutureTask<>(test);
new Thread(task,"testThread").start();
try {
    System.out.println("线程返回值为:"+task.get());
} catch (InterruptedException e) {
    e.printStackTrace();
} catch (ExecutionException e) {
    e.printStackTrace();
}

因为上边在call方法中定义的返回值类型为Integer，所以在FutureTask中定义泛型类型为Integer。

多线程的一些重要方法

  1. join方法

在调用join方法的线程运行完之后才会运行其他的线程。

Thread thread1 = new ThreadTest("thread1");
thread1.start();
//try {
//    thread1.join();
//} catch (InterruptedException e) {
//    e.printStackTrace();
//}
Thread thread2 = new ThreadTest("thread2");
thread2.start();

这样会使thread1执行完之后再执行thread2

  1. interrupt方法

该方法不会使线程立即终止

public class InterruptTest extends Thread{
    @Override
    public void run() {
        int n = 0;
        while(!isInterrupted()){
            n++;
            System.out.println(n);
        }
    }
}

Thread thread3 = new InterruptTest();
thread3.start();
try {
    Thread.sleep(1);
} catch (InterruptedException e) {
    e.printStackTrace();
}
thread3.interrupt();
try {
    thread3.join();
} catch (InterruptedException e) {
    e.printStackTrace();
}
System.out.println("end");

在测试方法中每次都会判断线程是否interrupt,只有响应这个线程才能终止。

注意：interupt只是将中断码设为true，需要代码实行逻辑。

线程池

多线程虽然可以提高运行效率，但是如果开启的线程过多，也会造成负面效应，因为在线程创建和销毁的时候会占用很多的CPU资源，造成资源浪费。

这样就引入了线程池–可以复用一组线程。

线程池内部维护了若干个线程，没有任务的时候，这些线程都处于等待状态。如果有新任务，就分配一个空闲线程执行。如果所有线程都处于忙碌状态，新任务要么放入队列等待，要么增加一个新线程进行处理。

Java标准库提供了ExecutorService接口表示线程池

因为ExecutorService只是接口，Java标准库提供的几个常用实现类有：

  • FixedThreadPool：线程数固定的线程池；
  • CachedThreadPool：线程数根据任务动态调整的线程池； 可以设置线程数的范围
  • SingleThreadExecutor：仅单线程执行的线程池。

这是一个线程实现类

public class Task implements Runnable{
    private String taskName;

    public Task(String taskName){
        this.taskName = taskName;
    }

    @Override
    public void run() {
        System.out.println("start task" + " " + taskName);
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        System.out.println("end task "+ taskName);
    }
}

使用线程池调用

public class ThreadPoolTest {
    public static void main(String [] args){
        ExecutorService service = Executors.newFixedThreadPool(4);
        for(int i = 0;i < 6; i++){
            service.submit(new Task(""+i));
        }
        service.shutdown();
    }
}

观察运行结果就会发现会先运行前四个任务，等四个任务完成（也就是线程空闲后）之后才会运行后面的两个任务

需要反复执行的线程ScheduledThreadPool

ScheduledExecutorService scheduledExecutorService = Executors.newScheduledThreadPool(4);
scheduledExecutorService.schedule(new Task("one time"),1, TimeUnit.SECONDS);
scheduledExecutorService.scheduleAtFixedRate(new Task("fixed rate"),2,3,TimeUnit.SECONDS);
scheduledExecutorService.scheduleWithFixedDelay(new Task("fixed delay"),2,3,TimeUnit.SECONDS);
